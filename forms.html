Formulaires Headless

Le module Formulaires adopte une philosophie "Headless" : le plugin gère la configuration, la sécurité, la validation, le stockage et les emails. Vous avez une liberté totale pour construire le HTML comme vous le souhaitez.

Configuration dans le Lab

Rendez-vous dans Blocks Master Lab → Formulaires :

Onglet Champs

Définissez vos champs avec :

  • Clé : Identifiant technique utilisé dans le code (ex: email, sujet).
  • Label : Texte affiché dans les emails et les soumissions.
  • Type : text, textarea, email, tel, number, select, radio, checkbox.
  • Requis : Cochez pour rendre le champ obligatoire.
  • Options (pour select/radio) : JSON ["A","B","C"] ou liste A,B,C.

Onglet Emails Admin & Utilisateur

Configurez le sujet et le corps avec des placeholders dynamiques entourés d'accolades :

Sujet : Nouvelle demande de {prenom} {nom}

Corps :
Prénom  : {prenom}
Email   : {email}
Message : {message}
Reçu le : {date}

Intégration PHP — Exemple Complet

Pour afficher un formulaire dans un bloc PHP, utilisez les helpers API :

<?php $slug = 'contact'; ?>

<?php if (sm_form_has_success($slug)): ?>
    <div class="tw-bg-emerald-50 tw-border tw-border-emerald-200 tw-rounded-lg tw-p-4">
        <p class="tw-text-emerald-700 tw-font-medium">
            <?= sm_get_form_success_message($slug) ?>
        </p>
    </div>
<?php else: ?>

    <form method="POST" class="tw-space-y-4">
        <?php sm_form_nonce_field($slug); ?>

        <!-- Champ Texte -->
        <div>
            <label class="tw-block tw-text-sm tw-font-medium tw-text-slate-700">Nom</label>
            <input type="text" name="nom"
                   value="<?= esc_attr(sm_form_old_value($slug, 'nom')) ?>"
                   class="tw-mt-1 tw-block tw-w-full tw-border tw-rounded-md tw-px-3 tw-py-2">
            <?php sm_form_field_error($slug, 'nom'); ?>
        </div>

        <!-- Champ Email -->
        <div>
            <label class="tw-block tw-text-sm tw-font-medium tw-text-slate-700">Email</label>
            <input type="email" name="email"
                   value="<?= esc_attr(sm_form_old_value($slug, 'email')) ?>"
                   class="tw-mt-1 tw-block tw-w-full tw-border tw-rounded-md tw-px-3 tw-py-2">
            <?php sm_form_field_error($slug, 'email'); ?>
        </div>

        <button type="submit" class="tw-bg-indigo-600 tw-text-white tw-px-6 tw-py-2 tw-rounded-lg">
            Envoyer
        </button>
    </form>

<?php endif; ?>

Types spéciaux : textarea et select

🔴 IMPORTANT

Les champs textarea et select ne peuvent pas être rendus avec une balise <input>. Le handler backend reçoit bien les données, mais le navigateur ne les enverra pas si le markup HTML est incorrect.

Textarea — Champ texte long

<div>
    <label class="tw-block tw-text-sm tw-font-medium tw-text-slate-700">Message</label>
    <textarea name="message" rows="5"
              class="tw-mt-1 tw-block tw-w-full tw-border tw-rounded-md tw-px-3 tw-py-2"><?= esc_html(sm_form_old_value($slug, 'message')) ?></textarea>
    <?php sm_form_field_error($slug, 'message'); ?>
</div>
⚠️ ATTENTION

La valeur pour textarea se place entre les balises ouvrante et fermante, sans espace ni retour à la ligne avant/après. Un espace parasite sera affiché dans le champ.

Select — Liste déroulante

<div>
    <label class="tw-block tw-text-sm tw-font-medium tw-text-slate-700">Sujet</label>
    <select name="sujet" class="tw-mt-1 tw-block tw-w-full tw-border tw-rounded-md tw-px-3 tw-py-2">
        <option value="">-- Choisissez un sujet --</option>
        <?php foreach (sm_form_get_field_options($slug, 'sujet') as $option): ?>
            <option value="<?= esc_attr($option) ?>"
                    <?= selected(sm_form_old_value($slug, 'sujet'), $option, false) ?>>
                <?= esc_html($option) ?>
            </option>
        <?php endforeach; ?>
    </select>
    <?php sm_form_field_error($slug, 'sujet'); ?>
</div>

Flash Messages — Comment ça marche

SiteMaster n'utilise aucune session PHP (session_start()). Les erreurs et les anciennes valeurs sont transmises via des Transients WordPress avec un token unique dans l'URL.

Cycle PRG (Post/Redirect/Get)

1

L'utilisateur soumet le formulaire (POST).

2

Les erreurs + anciennes valeurs sont stockées dans un transient avec un token unique (5 min).

3

Redirection vers ?sm_form_token=xyz.

4

La page se recharge (GET). Les helpers lisent et effacent le transient en une seule lecture (one-shot).

💡 TIP

Ce mécanisme est 100% compatible avec Cloudflare, WP Rocket, LiteSpeed Cache et les hébergements qui n'autorisent pas les sessions PHP.

Mode Expert — Validation AJAX

Si vous gérez la soumission en JavaScript, vous pouvez réutiliser le moteur de validation de SiteMaster directement dans votre endpoint WordPress :

add_action('wp_ajax_nopriv_mon_form', 'mon_handler_ajax');
function mon_handler_ajax(): void {
    check_ajax_referer('sm_submit_form_contact', 'sm_form_nonce');

    $result = sm_form_validate('contact', $_POST);

    if (!$result['success']) {
        // $result['errors'] : tableau [field_key => message d'erreur]
        wp_send_json_error($result['errors']);
    }

    // $result['data'] : données sanitisées et prêtes à l'insertion
    sm_insert_form_entry($form->id, $result['data']);
    wp_send_json_success(['message' => 'Formulaire envoyé avec succès']);
}
