Core SiteMaster — Items & Champs

Ces fonctions s'utilisent dans vos blocs sur les pages qui affichent un Item dynamique (via le Routage URL). Elles lisent automatiquement le contexte courant sans que vous ayez à passer un ID.

sm_get_current_value( string $key ) Core

Retourne la valeur d'un champ personnalisé de l'Item en cours. $key correspond à la clé unique définie lors de la configuration du champ dans le Groupe.

<h2><?= esc_html(sm_get_current_value('titre_section')) ?></h2>
<p><?= esc_html(sm_get_current_value('prix')) ?> € / mois</p>
sm_get_current_term_name() Core

Retourne le titre de l'Item en cours de lecture. Pratique pour les balises <h1> de vos templates.

<h1 class="tw-text-4xl tw-font-bold">
    <?= esc_html(sm_get_current_term_name()) ?>
</h1>
sm_get_current_content() Core

Retourne le contenu Gutenberg complet de l'Item (HTML déjà généré), images et blocs inclus.

<div class="tw-prose tw-max-w-none">
    <?= sm_get_current_content() ?>
</div>
sm_get_current_term_id() Core

Retourne l'ID WordPress (entier) de l'Item actuel. Utile pour des requêtes personnalisées ou des hooks.

<?php $post_id = sm_get_current_term_id(); ?>
<?php $galerie = get_post_meta($post_id, '_galerie_photos', true); ?>
sm_get_current_values() : array Core

Retourne tous les champs structurés de l'Item courant sous forme d'un tableau [field_key => valeur]. Utile pour un affichage en boucle ou pour debugger.

<?php $values = sm_get_current_values(); ?>
<?php foreach ($values as $key => $val): ?>
    <p><strong><?= esc_html($key) ?></strong> : <?= esc_html($val) ?></p>
<?php endforeach; ?>
sm_get_terms_for_path( string $path ) : array Core

Retourne tous les Items publiés d'un groupe en passant son listing_path. Indispensable pour construire une page de listing manuellement dans un bloc.

<?php $services = sm_get_terms_for_path('services'); ?>

<div class="tw-grid tw-grid-cols-3 tw-gap-6">
    <?php foreach ($services as $service): ?>
        <a href="<?= esc_url(sm_get_term_url($service)) ?>">
            <?= esc_html($service->post_title) ?>
        </a>
    <?php endforeach; ?>
</div>
sm_get_term_url( WP_Post $term ) : string Core

Construit l'URL canonique d'un Item (ex: /services/nettoyage/). Utilise le listing_path du groupe et le slug de l'Item.

<a href="<?= esc_url(sm_get_term_url($service)) ?>" class="tw-font-medium">
    <?= esc_html($service->post_title) ?>
</a>

SiteMaster Forms — Helpers Formulaires

Ces fonctions sont disponibles dès lors que le module Forms est actif. Elles n'ont aucune dépendance avec le module Core.

sm_form_nonce_field( string $slug ) Forms

Obligatoire. Affiche les champs cachés de sécurité (nonce + slug). À placer dans la balise <form>.

<form method="POST">
    <?php sm_form_nonce_field('contact'); ?>
    ...
</form>
sm_form_has_success( string $slug ) : bool Forms

Retourne true si le formulaire vient d'être soumis avec succès. Détecte le paramètre ?sm_form_success=slug dans l'URL.

<?php if (sm_form_has_success('contact')): ?>
    <div class="tw-bg-emerald-50 tw-p-4 tw-rounded-lg">
        <?= sm_get_form_success_message('contact') ?>
    </div>
<?php endif; ?>
sm_form_has_errors( string $slug ) : bool Forms

Retourne true si des erreurs de validation existent pour ce formulaire (flash transient actif). Utile pour afficher un message d’erreur global en haut du formulaire.

<?php if (sm_form_has_errors('contact')): ?>
    <div class="tw-bg-red-50 tw-border tw-border-red-200 tw-rounded-lg tw-p-4">
        <p class="tw-text-red-700">Certains champs sont invalides.</p>
    </div>
<?php endif; ?>
sm_form_old_value( string $slug, string $key ) : string Forms

Retourne la valeur saisie lors d'une soumission invalide. Utilisez-le dans l'attribut value pour ne pas perdre la saisie de l'utilisateur.

<input type="text" name="nom"
       value="<?= esc_attr(sm_form_old_value('contact', 'nom')) ?>">

<textarea name="message"><?= esc_html(sm_form_old_value('contact', 'message')) ?></textarea>
sm_form_field_error( string $slug, string $key, string $class = 'sm-field-error' ) Forms

Affiche directement un <span> contenant le message d'erreur pour un champ spécifique. Retourne rien si aucune erreur.

<?php sm_form_field_error('contact', 'email'); ?>
<!-- Génère : <span class="sm-field-error">Format invalide</span> -->

<!-- Avec classe personnalisée -->
<?php sm_form_field_error('contact', 'email', 'tw-text-red-600 tw-text-sm'); ?>
sm_form_get_field_options( string $slug, string $key ) : array Forms

Retourne la liste des options configurées pour un champ select, radio ou checkbox.

<select name="sujet">
    <?php foreach (sm_form_get_field_options('contact', 'sujet') as $opt): ?>
        <option value="<?= esc_attr($opt) ?>"><?= esc_html($opt) ?></option>
    <?php endforeach; ?>
</select>
sm_form_validate( string $slug, array $data ) : array Forms

Moteur de validation universel. Valide et sanitise un tableau de données selon la configuration du formulaire. Retourne un tableau structuré.

$result = sm_form_validate('contact', $_POST);

// $result = [
//   'success' => false,
//   'errors'  => ['email' => 'Format invalide'],
//   'data'    => ['nom' => 'Jean', 'email' => '']
// ]

if ($result['success']) {
    sm_insert_form_entry($form->id, $result['data']);
}

Emails & Délivrabilité SMTP

💡 TIP

SiteMaster utilise wp_mail() pour tous ses envois (notifications admin et emails de confirmation utilisateur). Par défaut, wp_mail() envoie via le serveur mail de votre hébergeur, ce qui peut aboutir en SPAM.

Pour une délivrabilité optimale en production, installez un plugin SMTP :

  • • FluentSMTP (recommandé, gratuit)
  • • WP Mail SMTP
  • • Post SMTP

Configurez-le avec SendGrid, Mailgun, Amazon SES ou Gmail. Aucune configuration supplémentaire n'est requise dans SiteMaster — il utilisera automatiquement le SMTP configuré.

Bonnes pratiques de Sécurité

Toutes les valeurs retournées par les helpers SiteMaster sont sanitisées côté serveur. Cependant, lors de l'affichage dans vos templates HTML, appliquez toujours les fonctions d'échappement WordPress :

Contexte d'affichage Fonction à utiliser
Texte simple dans le HTML esc_html()
URL dans href ou src esc_url()
Valeur dans un attribut HTML esc_attr()
Contenu Gutenberg / HTML riche wp_kses_post() 
<!-- Correct -->
<h1><?= esc_html(sm_get_current_term_name()) ?></h1>
<img src="<?= esc_url(sm_get_current_value('photo')) ?>" alt="">
<div><?= wp_kses_post(sm_get_current_content()) ?></div>
