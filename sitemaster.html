SiteMaster — Données & Modèles

SiteMaster transforme WordPress en un véritable CMS "Content-First". Créez des structures de données personnalisées, liez-les à des pages modèles, et bénéficiez d'un routage URL automatique sans une seule ligne de configuration.

Structure : Groupes, Champs & Items

SiteMaster repose sur trois concepts qui s'emboîtent naturellement :

1

Groupe de champs

Votre "Collection" ou type de contenu. Ex : Services, Projets, Membres de l'équipe. Un groupe peut avoir autant de champs que vous le souhaitez.

2

Champs personnalisés

Les données spécifiques à capturer. Types supportés : Texte, Texte Long, Image, URL, Nombre, Date, Email.

3

Items

Les enregistrements individuels. Chaque Item appartient à un Groupe. Ex : "Nettoyage de vitres" est un Item du groupe "Services".

Modèles & Routage Dynamique (URL Mapping)

C'est la fonctionnalité centrale de SiteMaster. Elle permet de créer des architectures "Listing / Détail" sans configuration manuelle.

Étape 1 — Créer une Page Modèle

Allez dans Pages Modèles et créez votre design dans l'éditeur Gutenberg standard. Intégrez vos blocs personnalisés pour afficher les données dynamiques via les helpers API.

Étape 2 — Configurer le Routage

Dans les réglages de votre Groupe de champs, renseignez :

  • Chemin de listing : Le préfixe de l'URL. Ex : services, portfolio.
  • Page Modèle : Le modèle Gutenberg créé à l'étape 1.

Étape 3 — Résultat automatique

Toute URL correspondant au pattern /{chemin}/{slug-item} est interceptée :

https://votre-site.com/portfolio/mon-projet-genial
 ↑ ↑
 chemin listing slug de l'Item

Voici ce qui se passe en coulisses :

  1. SiteMaster détecte l'URL et identifie l'Item mon-projet-genial.
  2. Il force WordPress à afficher votre Page Modèle configurée.
  3. Vos blocs PHP récupèrent les données de cet Item via les helpers API.
  4. Le visiteur voit la page du projet — sans aucun code de routing manuel.

Layouts de Template

Lors de la création d’une Page Modèle, vous choisissez le layout de rendu. Ce choix détermine comment le template s’insère dans votre thème WordPress.

🎨 Layout : Theme

Le template est rendu dans votre thème WordPress. Le header, footer et sidebar du thème sont présents. Idéal pour les pages de détail qui doivent conserver la navigation globale.

⬜ Layout : Blank

Le template est rendu dans une page vierge (HTML complet sans thème). Idéal pour des landing pages, pages de détail full-screen, ou des interfaces sans barre de navigation.

💡 TIP

Le layout Blank charge tout de même wp_head() et wp_footer(), donc vos scripts et styles WordPress (Tailwind, Lucide) sont disponibles. Il désactive uniquement le thème.

Gestion des Médias (Images)

SiteMaster est totalement intégré à la Bibliothèque de Médias WordPress.

  • Les champs de type "Image" utilisent le sélecteur natif WordPress — vos utilisateurs ont une expérience familière.
  • Côté code, SiteMaster retourne l'URL directe du média. Pas de tableaux complexes à déstructurer.
<?php $photo = sm_get_current_value('photo_couverture'); ?>

<?php if ($photo): ?>
 <img src="<?= esc_url($photo) ?>"
 alt="<?= esc_attr(sm_get_current_term_name()) ?>"
 class="tw-w-full tw-rounded-none tw-object-cover">
<?php else: ?>
 <div class="tw-bg-slate-200 tw-rounded-none tw-h-48"></div>
<?php endif; ?>

SEO & Performances

Puisque SiteMaster utilise de vraies Pages WordPress comme modèles :

  • La gestion du SEO (titres, méta-descriptions) reste standard — Yoast, RankMath, etc. fonctionnent nativement.
  • Les systèmes de cache (WP Rocket, Cloudflare) fonctionnent sans configuration.
  • Le contenu est indexable par les moteurs de recherche comme tout contenu WordPress natif.
🔴 IMPORTANT

SiteMaster masque les Items de l'interface standard de WordPress (show_in_menu = false) pour garder votre admin propre. Tout se gère depuis l'onglet Items dans votre Groupe de champs.
